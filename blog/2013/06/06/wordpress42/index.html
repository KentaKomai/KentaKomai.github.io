twitterのTLに流れてきたリンク

http://pastebin.com/EY3WQNzC

casehack.cではswitch-case文の面白い手法を紹介している。

#include <stdio.h>

int main() {
    int i=5;
    switch ( 2 ) {
        case 0:
            for (  i=0; i<10; i++ ) {
        case 1:
            printf("A i=%d\n",i);
        case 2:
            printf("B i*i=%d\n",i*i);
            };
        case 3:
            printf("done");
            break;
    }

    return 0;
}

上記のサンプルコードをコンパイルし、実行すると

/* OUTPUT
B i*i=25
A i=6
B i*i=36
A i=7
B i*i=49
A i=8
B i*i=64
A i=9
B i*i=81
done
*/

こちらが出力される、というもの。

ちょっと意味がわからないｗｗｗ

 

#include <stdio.h>

int main() {
    int i=5;
    switch ( 2 ) {
        case 0:
        printf("befor for\n");
            for (  i=0; i<10; i++ ) {
        printf("%d\n", i);
        case 1:
            printf("A i=%d\n",i);
        case 2:
            printf("B i*i=%d\n",i*i);
            };
        case 3:
            printf("done");
            break;
    }

    return 0;
}

流れを追っかけるためにprintfをcase:0の処理に注入。

この状態で実行する

B i*i=25
6
A i=6
B i*i=36
7
A i=7
B i*i=49
8
A i=8
B i*i=64
9
A i=9
B i*i=81
done

これが出力されました。

case 0: 内部の１行目のコードが実行されていないことがわかります。

以下が流れ

１．switch(2)なので case 2:のブロックが実行される

printf("B i*i=%d\n",i*i)

２．case 2:の最後にある } (中括弧)に対応しているfor文の頭に戻る。

for (  i=0; i<10; i++ ) {

３．switch-case文に入る前から i は5なので、あとはそのままi++されながら10になるまでループ

これによって、

for (  i=0; i<10; i++ ) {
        printf("%d\n", i);
        case 1:
            printf("A i=%d\n",i);
        case 2:
            printf("B i*i=%d\n",i*i);
            };

この箇所が４回繰り返される。

 

goto文のスコープを限定した局所的なものが、switch-case文って感じなのかな？？

これ使い道があるかは別として、面白かったので手元で再現してみました。
